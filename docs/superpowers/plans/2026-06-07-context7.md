# Context7 Plugin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `context7` plugin that installs the ctx7 CLI and find-docs skill for all agents via npx skills@latest.

**Architecture:** YAML plugin in `defaults.yaml` following the cocoindex-code pattern — npm global install + skills CLI add. Optional `CONTEXT7_API_KEY` for higher rate limits is set via `vm.env`, not plugin config.

**Tech Stack:** Go plugin system, bash setup scripts, bootstrap Docker tests

---

## Task 1: Plugin definition

**Files:**

- Modify: `internal/plugin/defaults.yaml`

- [x] **Step 1: Add context7 plugin block after cocoindex-code**

```yaml
context7:
  description: "Context7 — up-to-date library documentation via find-docs skill"
  dependencies:
    - mise-node
  skip_if: |
    command -v ctx7 >/dev/null 2>&1 && \
    find "$HOME" -path "*/skills/find-docs/SKILL.md" 2>/dev/null | grep -q .
  setup: |
    npm install -g ctx7@latest
    npx skills@latest add upstash/context7 --global --yes --agent '*' --skill find-docs
```
